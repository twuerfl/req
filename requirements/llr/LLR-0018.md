## Description

The tool shall generate coverage reports showing the percentage of requirements with implementations, tests, and traceability links.

## Rationale

Coverage metrics are essential for safety audits. They provide quantitative evidence of completeness and help identify gaps before certification reviews.

## Acceptance Criteria

- [ ] `req coverage` shows HLR and LLR coverage percentages
- [ ] Report shows LLR implementation rate
- [ ] Report shows LLR test rate
- [ ] `--format json` option for machine-readable output
- [ ] Exit code 1 in strict mode if coverage below threshold

## Implementation

### Text Output

```
Coverage Report
═══════════════

High-Level Requirements:
  Total:      11
  With LLR:   10 (90.9%)

Low-Level Requirements:
  Total:       16
  Implemented: 13 (81.2%)
  Tested:      0 (0.0%)
```
