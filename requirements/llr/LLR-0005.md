## Description

The system shall provide a `TraceGraph` that builds and traverses the full
HLR→LLR→TST→Code traceability graph, enabling gap detection, impact analysis,
traceability validation, and trace tree output.

## Rationale

Graph-based representation enables efficient gap detection, impact analysis, and
traceability validation required for safety standards. The graph is the authoritative
view of all requirement relationships and their coverage state.

## Acceptance Criteria

- [ ] Builds graph from cache data (requirements, links, code refs)
- [ ] Finds gaps: HLR without LLR, LLR without code ref, LLR without test
- [ ] Detects orphan code refs (code refs pointing to undefined requirement IDs)
- [ ] Validates all traceability links and reports broken references
- [ ] Provides impact analysis showing all requirements and files affected by a change
- [ ] Produces a human-readable trace tree for a single requirement (parent + children + code refs)
