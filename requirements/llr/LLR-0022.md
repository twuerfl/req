## Description

The `req_engine` crate contains all qualified business logic: the SQLite cache, code scanner, Markdown/JSON/ReqIF adapters, traceability graph, AI import, provenance tracking, and parser. All external callers access engine functionality exclusively through the `ReqEngine` facade struct.

## Implementation

- Location: `req_engine/src/`
- Entry point: `req_engine/src/lib.rs`
- Main facade: `req_engine/src/engine.rs` (`ReqEngine` struct)
- Depends on: `req_lib` (path dependency)
- Re-exports all `req_lib` public types so callers need only one dependency
- Modules: `adapter`, `ai_import`, `cache`, `config`, `engine`, `error`, `parser`, `provenance`, `scanner`, `trace`

## Acceptance Criteria

- [ ] All business logic operations (scan, coverage, gap detection, impact analysis, export) are accessible via `ReqEngine` methods
- [ ] `req_engine` re-exports all `req_lib` public types at its crate root
- [ ] No presentation logic (terminal colours, interactive prompts) exists in this crate
- [ ] `cargo clippy -p req_engine` reports zero warnings
- [ ] `cargo test -p req_engine` passes
