## Description

The system shall provide a Requirement struct with ID, type, title, text, status, parent, and metadata fields.

## Rationale

A structured data model enables consistent handling of requirements across all adapters and ensures data integrity. Without this, requirement data would be inconsistent across different formats.

## Acceptance Criteria

- [ ] Requirement struct exists with all required fields
- [ ] ID format validation (HLR-XXXX, LLR-XXXX, TST-XXXX)
- [ ] Auto-generation of sequential IDs
