## Description

The system shall implement incremental scanning by storing a SHA-256 hash of each
scanned source file in the cache. On subsequent scans, only files whose hash has changed
since the last scan are re-parsed; unchanged files are skipped.

## Rationale

Large repositories can contain thousands of source files. Re-parsing every file on each
invocation would make `req scan` prohibitively slow. Hash-based change detection limits
re-parsing to only the files that have actually changed, keeping scan latency proportional
to the size of the diff rather than the size of the repository.

## Acceptance Criteria

- [ ] File SHA-256 hash is stored in the `file_hashes` table after each successful scan
- [ ] Files whose stored hash matches the current on-disk hash are skipped during scanning
- [ ] Files whose stored hash differs (or are absent from the table) are fully re-parsed
- [ ] `req scan --clear` deletes all cached hashes and forces a full re-scan
- [ ] Incremental scan produces the same traceability result as a full `--clear` scan on the same repository state
