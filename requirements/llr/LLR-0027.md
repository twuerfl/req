## Description

The `req_engine` shall contain an `audit::mutation` module that parses the JSON output of
`cargo mutants --json` and maps per-file mutation scores to the LLR tags found in the
overlapping source line ranges, producing a `MutationScore` per LLR.

`req` does not invoke `cargo mutants` itself. The user runs it independently and passes the
resulting JSON file as input. This keeps `req` usable in air-gapped environments.

## Implementation Notes

- Module: `req_engine/src/audit/mutation.rs`
- No compile-time dependency on `cargo-mutants`
- Uses `serde_json` (already present) to parse the JSON
- Entry point: `pub fn parse_mutants_report(json_path: &Path, code_refs: &HashMap<String, Vec<CodeRef>>) -> Result<Vec<MutationScore>>`

## `cargo mutants --json` Schema (relevant fields)

```json
[
  {
    "file": "src/engine.rs",
    "line": 42,
    "kind": "ReplaceWithDefault",
    "outcome": "Caught" | "Missed" | "Unviable" | "Timeout"
  }
]
```

## Correlation Algorithm

For each mutant record:
1. Normalise `file` to an absolute path.
2. Find all `CodeRef` entries where `code_ref.file == file` and `line` falls within
   `[code_ref.line, code_ref.line_end.unwrap_or(code_ref.line + 1))`.
3. For each matching `req_id`, accumulate into `MutationScore`.
4. `Unviable` mutants count in `mutants_total` but not in `caught` or `missed`.
5. Mutants matching no `CodeRef` range are counted in a separate `untagged_mutants` total.

## User Workflow

```bash
cargo mutants --json > .req/mutants-report.json
req audit mutation --report .req/mutants-report.json
```

## Acceptance Criteria

- [ ] Parses valid `cargo mutants` JSON output without panics
- [ ] A mutant whose `file`+`line` overlaps a `CodeRef` range is attributed to that `req_id`
- [ ] A mutant outside all `CodeRef` ranges increments the `untagged_mutants` counter
- [ ] `Unviable` mutants appear in `mutants_total` but not in `caught` or `missed`
- [ ] Malformed or empty JSON returns `Error::Parse` with a descriptive message
- [ ] `MutationScore` contains: `req_id`, `mutants_total`, `caught`, `missed`, `score_percent`
