## Description

The system shall record the originating file path and its SHA-256 hash for every
requirement created via an import command, and shall detect during incremental scanning
whether that source file has since changed or been deleted.

## Rationale

Import commands (`req import`, `req import-ai`, `req import-reqif`) write requirements to
disk but currently record no link back to their originating file. If the source document
is later modified or deleted, the imported requirements silently become inconsistent with
their origin. Staleness detection surfaces this divergence during the normal scan+check
workflow, prompting the engineer to re-import or explicitly accept the divergence, which
is required for traceability in safety-critical projects.

## Schema Extension

A new `import_sources` table shall be added to `cache.db`:

```sql
CREATE TABLE IF NOT EXISTS import_sources (
    req_id       TEXT PRIMARY KEY,
    source_path  TEXT NOT NULL,
    sha256       TEXT NOT NULL,
    imported_at  TEXT NOT NULL   -- ISO-8601 UTC timestamp
);
```

## Staleness States

| State | Condition | Flag stored in cache |
|---|---|---|
| `import_stale` | Source file exists but SHA-256 differs from `sha256` | `stale` |
| `import_orphaned` | Source file path no longer exists on disk | `orphaned` |
| (clean) | SHA-256 matches or row absent | â€” |

A new `import_status` column is added to the `requirements` cache table to hold these
flags (`NULL` = not imported or clean).

## Acceptance Criteria

- [ ] `req import`, `req import-ai`, and `req import-reqif` each populate `import_sources`
      with `req_id`, absolute `source_path`, `sha256` of the source file at import time,
      and `imported_at` timestamp
- [ ] `req scan` (incremental pass) iterates every row in `import_sources` and:
      - Sets `import_status = 'stale'` on the requirement row when the file exists but
        its current SHA-256 differs from the stored hash
      - Sets `import_status = 'orphaned'` on the requirement row when the file is absent
      - Clears `import_status` to NULL when the current SHA-256 matches the stored hash
- [ ] `req check` reports every requirement with `import_status = 'stale'` or `'orphaned'`
      as a validation error and exits non-zero
- [ ] `req list` includes an `import_status` column in its output when any flagged
      requirements are present
- [ ] `req gaps` surfaces orphaned imports as unresolved gaps
- [ ] `req remove <ID>` (LLR-0035) deletes the corresponding `import_sources` row
- [ ] Re-running the original import command with the same `--output` path updates the
      `import_sources` row (upsert) and clears the `import_status` flag
- [ ] `req import` with a `--source <path>` override allows the stored source path to be
      updated without re-importing all requirements
