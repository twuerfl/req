## Description

The system shall provide a SQLite-based cache that is a non-authoritative performance
layer. Requirement files on disk are the sole authoritative store; the cache may be
deleted and fully rebuilt from the file system at any time without data loss.

## Rationale

SQLite provides fast local storage for incremental scanning and query performance.
Keeping the cache non-authoritative ensures that Git remains the single source of truth:
the repository state alone is sufficient to reconstruct all traceability data. Excluding
`cache.db` from version control prevents stale or machine-specific cache state from
entering the repository.

## Acceptance Criteria

- [ ] Schema with requirements, code_refs, links, file_hashes tables
- [ ] Upsert operations for requirements and code refs
- [ ] `cache.db` is listed in `.gitignore`; it is never committed to the repository
- [ ] Deleting `cache.db` and running `req scan` reproduces identical `req list` and `req coverage` output (rebuild idempotency)
