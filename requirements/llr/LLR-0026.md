## Description

The `req_engine` shall contain an `audit::triviality` module that reads the source lines
of a `CodeRef` span and applies hollow-body detection heuristics, returning a
`TrivialityFinding` per detected pattern.

## Implementation Notes

- Module: `req_engine/src/audit/triviality.rs`
- Entry point: `pub fn analyse_code_ref(path: &Path, line: usize, line_end: Option<usize>) -> Result<Vec<TrivialityFinding>>`
- Engine facade method: `pub fn audit_triviality(&self) -> Result<Vec<TrivialityReport>>`
- No new Cargo dependencies required (uses `regex`, already present)

## Detection Patterns

| Pattern | Description | Severity |
|---------|-------------|----------|
| P1 TautologicalAssertion | `assert!(true)`, `assert_eq!(x, x)` | Error |
| P2 StubBody | Sole body is `Ok(())`, `unimplemented!()`, `todo!()` | Warning |
| P3 ZeroSubstantiveLines | Tagged span has no non-comment, non-blank lines | Error |
| P4 SingleLiteralReturn | Function returns only a bare literal (`true`, `0`, `""`) | Warning |
| P5 EmptyTestBody | `#[test]` function with no assertions and no function calls | Error |
| P6 ArgumentEchoAssertion | `assert_eq!(f(x), x)` — output mirrors input | Info |

A line is "substantive" if it contains at least one of: `(`, `[`, `match`, `if`, `for`,
`while`, `let`, `->`.

Files in `build.rs`, `benches/`, or `examples/` directories are skipped.
Functions annotated with `#[allow(dead_code)]` have P2 downgraded from Warning to Info.

## Acceptance Criteria

- [ ] `analyse_code_ref` detects P1 (tautological assertion) in test spans and returns Error severity
- [ ] `analyse_code_ref` detects P3 (zero substantive lines) and returns Error severity
- [ ] `analyse_code_ref` on a legitimate multi-line function returns no findings
- [ ] `analyse_code_ref` detects P2 (`Ok(())` sole body) and returns Warning severity
- [ ] Files in `examples/` and `benches/` directories produce no findings
- [ ] `#[allow(dead_code)]` on the tagged function downgrades P2 from Warning to Info
- [ ] `audit_triviality()` aggregates findings from all cached `CodeRef`s into `TrivialityReport`s
