## Description

The system shall provide a safe import mechanism for AI-generated requirement suggestions that enforces human review before any requirement becomes approved.

## Rationale

AI-generated requirements can accelerate reverse engineering but must never bypass human review. Safety-critical systems require human accountability. The tool acts as a gatekeeper, not an approver.

## Acceptance Criteria

- [ ] `req import-ai <file>` command accepts AI suggestion JSON
- [ ] All imported requirements have status forced to `draft`
- [ ] Validation rejects suggestions with invalid parent IDs
- [ ] Validation rejects suggestions with duplicate IDs
- [ ] `--dry-run` flag previews imports without writing
- [ ] `--interactive` flag prompts for each suggestion
- [ ] Clear audit trail of AI origin in requirement metadata
