## Description

The `req_mcp` crate is the MCP server binary. It depends on `req_engine` for all business logic and on `rmcp` for the Model Context Protocol runtime. It is explicitly **not** in scope for tool qualification.

## Implementation

- Location: `req_mcp/src/`
- Binary entry point: `req_mcp/src/main.rs`
- Depends on: `req_engine` (path dependency), `rmcp` (version "1.2", features: `["server", "macros"]`), `tokio` (async runtime)
- Must not contain business logic; delegates all computation to `req_engine`
- Must not be referenced by `req_lib` or `req_engine`

## Acceptance Criteria

- [ ] Binary is named `req-mcp-server` and built from `req_mcp`
- [ ] `req_mcp/Cargo.toml` lists `rmcp` with `features = ["server", "macros"]`
- [ ] `cargo build -p req_mcp` succeeds (even while MCP tools are not yet implemented)
- [ ] `cargo clippy -p req_mcp` reports zero warnings
