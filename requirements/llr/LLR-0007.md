## Description

The system shall provide a `JsonAdapter` for reading and writing requirements in
machine-readable JSON format, used by `req export --format json` and
`req import --format json`.

## Rationale

JSON is the standard interchange format for CI pipelines and automation. The adapter
provides a stable roundtrip path: export → edit/generate → import. AI model interaction
is handled via the MCP server (LLR-0025), not via this adapter.

## Acceptance Criteria

- [ ] `req export --format json` outputs all requirements as a JSON array
- [ ] `req export --format json --id <ID>` outputs a single requirement
- [ ] `req import --format json <file>` imports requirements from a JSON array
- [ ] JSON roundtrip (export → import) preserves all fields without loss
