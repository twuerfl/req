## Description

The `req_cli` crate shall expose a top-level `audit` subcommand with six sub-subcommands,
dispatching to the `ReqEngine` audit methods introduced by LLR-0026 through LLR-0031.

## CLI Interface

```text
req audit triviality
    [-i|--id <ID>]        Filter to a single LLR (default: all LLRs)
    [--format text|json]  Output format (default: text)

req audit criteria <ID>
    (required positional)  The LLR to inspect

req audit mutation
    <REPORT>              Path to cargo mutants --json output file (positional)
    [--format text|json]

req audit coverage
    <REPORT>              Path to cargo llvm-cov --json output file (positional)
    [--format text|json]

req audit export-context <ID>
    (required positional)      The LLR to bundle
    [--mutation <PATH>]        Path to cargo-mutants JSON report
    [--coverage <PATH>]        Path to llvm-cov JSON report
    [-o|--output <PATH>]       Write bundle to file; defaults to stdout

req audit independence
    [-i|--id <ID>]        Filter to a single LLR (default: all)
    [--format text|json]
    (prints a warning and exits 0 when git feature is absent)
```

## Implementation Notes

- `Commands::Audit { command: AuditCommands }` variant in `req_cli/src/cli.rs`
- `AuditCommands` enum with one variant per subcommand
- `cmd_audit` dispatches to per-subcommand handlers in `req_cli/src/cli.rs`
- Text output formatting via helper functions in `req_cli/src/output.rs`
- All JSON output is produced by `serde_json::to_string_pretty` on the engine return value
- Global `--strict` flag causes non-zero exit on errors (applies to all commands)

## Acceptance Criteria

- [ ] `req audit triviality` runs without panic and exits 0 when no findings
- [ ] `req audit triviality --format json` output is valid JSON deserialising to an array
- [ ] `req audit mutation --report <missing-file>` exits non-zero with an error message
- [ ] `req audit export-context <ID>` writes a valid `AuditBundle` JSON to stdout
- [ ] `req audit independence` prints a warning (not error) when `git` feature is absent
- [ ] All six subcommands are listed in `req audit --help`
