## Description

The `req_cli` crate is the user-facing binary. It handles CLI argument parsing, terminal output formatting (colours, progress bars), and git hook installation. It is explicitly **not** in scope for tool qualification — all safety-relevant logic resides in `req_engine`.

## Implementation

- Location: `req_cli/src/`
- Binary entry point: `req_cli/src/main.rs`
- CLI definition: `req_cli/src/cli.rs` (clap derive)
- Output formatting: `req_cli/src/output.rs`
- Git hooks: `req_cli/src/hooks.rs`
- Depends on: `req_engine` (path dependency), `clap`, `colored`, `indicatif`, `anyhow`, `serde_json`
- Must not contain business logic; delegates all computation to `req_engine`

## Acceptance Criteria

- [ ] Binary is named `req` and built from `req_cli`
- [ ] All CLI commands delegate to `ReqEngine` methods; no requirement parsing or traceability logic in this crate
- [ ] `cargo clippy -p req_cli` reports zero warnings
- [ ] `cargo build -p req_cli` produces a working binary
