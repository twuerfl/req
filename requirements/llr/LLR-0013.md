## Description

The system shall track provenance of all requirement changes and protect sensitive operations with authentication, enabling external enforcement mechanisms to validate change integrity.

## Rationale

**Key Insight**: The tool cannot enforce its own restrictions. If an AI agent can use the tool, it can bypass tool-level locks. 

**Solution**: 
1. Track provenance for external validation (Git hooks, CI)
2. Protect sensitive operations with password/token
3. Direct file editing prevention is handled externally (outside tool scope)

## Acceptance Criteria

- [ ] Every tool-created/modified requirement has `tool_version` attribute
- [ ] `req check --provenance` validates all requirements have tool provenance
- [ ] Sensitive operations require `--token` or `REQ_TOKEN` env var
- [ ] Pre-commit hook template validates provenance
- [ ] Export provenance report for audit

## Protected Operations

Operations requiring authentication:
- `req import-ai` - AI suggestion import
- `req new --status approved` - Creating approved requirements
- `req import` - Bulk import

Operations NOT requiring authentication:
- `req scan` - Read-only code scan
- `req coverage` - Read-only report
- `req gaps` - Read-only analysis
- `req check` - Read-only validation
- `req export` - Read-only export

## Enforcement Model

```
┌─────────────────────────────────────────────────────────────┐
│  External Enforcement (Outside Tool Scope)                  │
│  - File permissions (read-only on requirements/)            │
│  - Git hooks (pre-commit validation)                        │
│  - CI/CD pipelines                                          │
│  - Code review process                                      │
└─────────────────────────────────────────────────────────────┘
                           │ validates
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  Tool: Provenance + Auth                                    │
│  - Records tool_version on every change                     │
│  - Token-protects sensitive operations                      │
│  - Provides check --provenance for validation               │
└─────────────────────────────────────────────────────────────┘
```

## Token Configuration

Token can be provided via:
1. `--token` command line argument
2. `REQ_TOKEN` environment variable
3. `.req/config.toml` (for local development, not recommended for CI)
