## Description

The tool shall generate impact analysis reports showing which files, tests, and other requirements would be affected by a change to a specific requirement.

## Rationale

Impact analysis is critical for change management in safety-critical systems. It helps assess the cost and risk of changes before implementation, ensuring all affected components are identified for re-testing.

## Acceptance Criteria

- [ ] `req impact <ID>` shows affected requirements
- [ ] Report shows affected source files
- [ ] Report shows affected tests
- [ ] Supports both upstream (parents) and downstream (children) impact
- [ ] JSON output option for CI integration

## Implementation

### Text Output

```
Impact Analysis for LLR-0001
═══════════════════════════

Affected requirements (3):
  - HLR-0001 (parent)
  - TST-0001 (test)
  - TST-0002 (test)

Affected files (2):
  - src/adc.c
  - src/filter.c

Affected tests (2):
  - TST-0001
  - TST-0002
```

### JSON Output

```json
{
