## Description

The system shall provide a `req remove <ID>` command that deletes a requirement file
from disk and purges all associated cache entries in a single atomic operation.

## Rationale

Requirement lifecycle management requires a controlled deletion path. Without it, deleted
files leave stale rows in `cache.db`, causing false positives in gap detection, coverage
reports, and traceability checks. A dedicated command ensures the file system and the
cache stay consistent and creates an explicit audit event in Git history.

## Acceptance Criteria

- [ ] `req remove <ID>` locates the `.md` file for the given requirement ID and deletes it
      from disk, then purges all rows referencing that ID from the `requirements`,
      `code_refs`, `links`, `file_hashes`, and `import_sources` cache tables
- [ ] A confirmation prompt is shown before the file is deleted; `--yes` (or `-y`) skips
      the prompt for scripted use
- [ ] `--cache-only` flag purges the cache entries without deleting the file on disk
      (recovery / repair scenario)
- [ ] If the ID does not exist in the cache or on disk, the command exits non-zero and
      prints an actionable error to stderr
- [ ] If the requirement has live child requirements (LLRs whose `parent:` field
      references it, or TSTs whose `parent:` field references it) or active code
      references in the cache, a warning is printed to stderr; with `--strict` this
      becomes a fatal error and the removal is aborted
- [ ] `--force` bypasses the dependency check and proceeds with removal even when
      dependents exist; it does not bypass the confirmation prompt
- [ ] The command is listed in `req --help` and has correct `req remove --help` output
- [ ] LLR-0006 command surface table is updated to include `remove <ID>`
