## Description

The `req_engine` shall contain an `audit::coverage_map` module that parses the JSON output
of `cargo llvm-cov --json` and maps LLVM segment line-hit data to LLR-tagged `CodeRef`
ranges, producing a `LineCoverageScore` per LLR.

`req` does not invoke `cargo llvm-cov` itself. The user runs it independently and passes
the resulting JSON file as input.

## Implementation Notes

- Module: `req_engine/src/audit/coverage_map.rs`
- No compile-time dependency on `cargo-llvm-cov`
- Uses `serde_json` (already present)
- Entry point: `pub fn parse_llvm_cov_report(json_path: &Path, code_refs: &HashMap<String, Vec<CodeRef>>) -> Result<Vec<LineCoverageScore>>`

## LLVM Coverage JSON Schema (relevant structure)

```json
{
  "data": [{
    "files": [{
      "filename": "src/engine.rs",
      "segments": [[line, col, exec_count, has_count, is_region_entry], ...]
    }]
  }]
}
```

A segment `[line, col, exec_count, ...]` defines coverage from that point until the next
segment. A line inside a tagged range where the last applicable segment has `exec_count == 0`
is counted as missed.

## Correlation Algorithm

For each `CodeRef` range `[line_start, line_end)`:
1. Filter `data[0].files` to the matching filename (normalise path separators).
2. For each integer line `L` in `[line_start, line_end)`:
   - Find the last segment with `segment[0] <= L`.
   - `exec_count > 0` → line hit; `exec_count == 0` → line missed.
3. Aggregate into `LineCoverageScore { req_id, file, lines_total, lines_hit, hit_percent }`.

## User Workflow

```bash
cargo llvm-cov --json --output-path .req/coverage-report.json
req audit coverage --report .req/coverage-report.json
```

## Acceptance Criteria

- [ ] Reads and parses LLVM coverage JSON format without panics
- [ ] A line with `exec_count == 0` inside a `CodeRef` range is counted as missed
- [ ] A line outside all `CodeRef` ranges is ignored
- [ ] Path separators are normalised before filename comparison (Windows/Unix)
- [ ] Malformed JSON returns `Error::Parse` with a descriptive message
- [ ] `LineCoverageScore` contains: `req_id`, `file`, `lines_total`, `lines_hit`, `hit_percent`
