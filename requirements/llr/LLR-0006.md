## Description

The system shall provide a clap-based CLI (`req`) with the following subcommands:

| Command | Description | Ref |
| ------- | ----------- | --- |
| `init` | Initialise a new requirements project | HLR-0002 |
| `new` | Create a new requirement | HLR-0002 |
| `scan` | Scan source code for requirement tags | LLR-0002 |
| `trace <ID>` | Show traceability tree for a requirement | LLR-0005 |
| `coverage` | Show coverage statistics | LLR-0018 |
| `gaps` | Show gaps in traceability | LLR-0005 |
| `check` | Validate all traceability links | LLR-0005 |
| `impact <ID>` | Impact analysis for a requirement | LLR-0020 |
| `export` | Export requirements (json, ai-context, markdown) | LLR-0017 |
| `import` | Import requirements (json, markdown) | LLR-0003 |
| `import-reqif` | Import from ReqIF file (requires reqif feature) | LLR-0009 |
| `export-reqif` | Export to ReqIF file (requires reqif feature) | LLR-0009 |
| `list` | List all requirements, optionally filtered by type | HLR-0002 |
| `hooks install\|uninstall` | Manage pre-commit hook | LLR-0011 |
| `ci <type>` | Generate CI workflow files | LLR-0010 |
| `import-ai` | Import AI-generated requirement suggestions | LLR-0012 |
| `check-provenance` | Check provenance of all requirements | LLR-0013 |
| `migrate` | Upgrade old requirement files to current schema | LLR-0016 |
| `audit` | AI agent output integrity auditing (6 subcommands) | LLR-0032 |
| `remove <ID>` | Remove a requirement file from disk and purge cache entries | LLR-0035 |

## Rationale

CLI-first design enables automation, CI integration, and scriptability. Git-like command
structure provides familiar UX for developers. Commands are additive — new commands are
added as new LLRs are introduced; this LLR tracks the complete command surface.

## Acceptance Criteria

- [ ] clap-based CLI with subcommands
- [ ] All commands listed in the table above are present in `req --help`
- [ ] `req <cmd> --help` shows correct arguments for each command
- [ ] Consistent exit codes for CI integration
- [ ] Strict mode (`--strict`) causes non-zero exit on warnings
- [ ] Global `--base <DIR>` and `--format text|json` options apply to all commands
