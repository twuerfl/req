## Description

The tool shall export requirements in JSON and Markdown formats for CI integration
and audit documentation. AI model interaction uses the MCP server (LLR-0025) instead.

## Rationale

JSON enables machine-readable exports for CI pipelines and downstream tooling.
Markdown provides human-readable documentation that can be versioned in Git.
The `ai-context` bulk-dump format is superseded by the MCP server, which provides
structured, queryable access to the same data.

## Acceptance Criteria

- [ ] `req export --format json` outputs all requirements as a JSON array
- [ ] `req export --format markdown` outputs all requirements as Markdown
- [ ] Single requirement export via `--id` option applies to both formats
- [ ] Output is written to stdout by default; `--output <file>` writes to a file

## Implementation

```bash
req export --format json --output requirements.json
req export --format json --id HLR-0001
req export --format markdown --output requirements.md
```
