## Description

The tool shall gracefully handle repositories created with older versions, automatically migrating data or providing clear migration paths.

## Rationale

Long-lived repositories may have been created with older tool versions. Breaking compatibility would orphan these repositories and force manual migration. Graceful degradation ensures continuity.

## Acceptance Criteria

- [ ] Import accepts files without `schema_version` (assumes 1.0.0)
- [ ] Import accepts files without `tool_version` (logs warning)
- [ ] Missing optional fields get default values
- [ ] Unknown fields are preserved in attributes (not dropped)
- [ ] Migration command `req migrate` to upgrade old files
- [ ] Migration is idempotent (safe to run multiple times)

## Implementation

### Import Tolerance

```rust
fn parse_requirement(value: Value) -> Result<Requirement> {
    // Schema version defaults to 1.0.0 if missing
    let schema_version = value.get("schema_version")
        .and_then(|v| v.as_str())
        .unwrap_or("1.0.0");
    
    // Unknown fields go to attributes
    let known_fields = ["id", "type", "title", "text", "status", "parent"];
    for (key, value) in value.as_object().unwrap() {
        if !known_fields.contains(&key.as_str()) {
            req.attributes.insert(key.clone(), value.to_string());
        }
    }
}
```

### Migration Command

```bash
req migrate --dry-run  # Preview changes
req migrate            # Upgrade all files to current schema
```
