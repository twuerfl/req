## Description

The tool shall generate traceability graph reports showing the complete chain from HLR to LLR to code implementation.

## Rationale

Traceability graphs are required for safety audits to demonstrate complete requirement decomposition. They provide visual evidence that every HLR is traced to implementation.

## Acceptance Criteria

- [ ] `req trace <ID>` shows traceability tree for a requirement
- [ ] `req gaps` shows all traceability gaps
- [ ] Tree shows parent → children → code refs
- [ ] Identifies orphan requirements (no parent/children)
- [ ] Identifies broken links (referenced ID not found)
