## Description

All `req` commands that produce structured output shall be deterministic: identical
repository state shall always produce byte-identical output, regardless of execution
order, platform, or environment.

## Rationale

CI pipelines compare outputs across runs. AI models process structured output and must
not be confused by spurious diffs. Auditors require that reports can be reproduced from
the same repository state. Non-determinism (random ordering, wall-clock timestamps,
machine-specific paths) breaks all three use cases.

## Acceptance Criteria

- [ ] `req list --format json` output is sorted by requirement ID and is stable across repeated invocations on the same repository state
- [ ] `req coverage --format json` output contains no wall-clock timestamps or process IDs
- [ ] `req gaps --format json` and `req check --format json` output lists are sorted
- [ ] File paths in structured output use forward slashes and are relative to `--base` (platform-independent)
- [ ] `req export --format json` output is identical for identical repository state (no generated-at timestamp in the requirements array)
