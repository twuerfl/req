## Description

The `req_engine` shall produce a structured JSON context bundle (`AuditBundle`) for a
given LLR that contains everything an independent LLM reviewer needs to judge whether
the implementation and tests genuinely satisfy the acceptance criteria.

`req` never makes network calls. The bundle is the input payload for an external LLM
invocation by the user or a CI pipeline step.

## Implementation Notes

- New types in `req_lib/src/audit.rs`: `AuditBundle`, `AuditCriterion`, `SourceSpan`,
  `AuditTrivialityFinding`, along with a compile-time constant `AUDIT_PROMPT_HINT`
- Engine method: `pub fn audit_export(llr_id: &str, mutation_path: Option<&Path>, coverage_path: Option<&Path>) -> Result<AuditBundle>`
- Source text is limited to the tagged span lines (not the entire file)
- Files that cannot be read produce a warning entry in the bundle rather than a hard error

## AuditBundle Structure

```
schema_version:          String  ("2.0.0")
tool_version:            String
generated_at:            String  (ISO 8601)
llr:                     Requirement
hlr:                     Option<Requirement>
acceptance_criteria:     Vec<AuditCriterion>
implementation_spans:    Vec<SourceSpan>
test_spans:              Vec<SourceSpan>
triviality_findings:     Vec<TrivialityFinding>
mutation_score:          Option<MutationScore>
line_coverage_score:     Option<LineCoverageScore>
criterion_coverage:      Vec<CriterionStatus>
prompt_hint:             String   (= AUDIT_PROMPT_HINT constant)
```

`AUDIT_PROMPT_HINT` is a static string constant in `req_lib` instructing the LLM to:
1. Read each `acceptance_criteria` item.
2. Find corresponding `test_spans` via `criterion_coverage`.
3. Determine whether the test exercises the criterion (not just a tautology).
4. Review `implementation_spans` for `triviality_findings`.
5. Identify unlinked criteria (`linked: false`).
6. Return structured JSON: `{ "verdict": "pass"|"fail"|"needs_review", "findings": [...] }`.

The prompt is a static constant to keep the qualified component deterministic and auditable
as part of tool-qualification evidence.

## Acceptance Criteria

- [ ] `audit_export` collects all fields without external tools when optional paths are absent
- [ ] `source_text` in each `SourceSpan` contains only the lines of the tagged span
- [ ] An unreadable source file adds a warning string to the bundle instead of returning an error
- [ ] The bundle serialises and deserialises round-trip via `serde_json` without data loss
- [ ] `prompt_hint` is non-empty and references all six review tasks listed above
- [ ] `schema_version` is `"2.0.0"` and `generated_at` is a valid ISO 8601 timestamp
