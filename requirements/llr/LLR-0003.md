## Description

The system shall provide a MarkdownAdapter that reads and writes requirements from/to Markdown files with YAML frontmatter.

## Rationale

Markdown with YAML frontmatter is human-readable, Git-friendly, and serves as the native storage format. This enables version control and easy editing.

## Acceptance Criteria

- [ ] Parser extracts YAML frontmatter and body text
- [ ] Writer generates valid Markdown with frontmatter
- [ ] Writer includes `tool_version` field in frontmatter (implements LLR-0013 provenance)
- [ ] Parser tolerates `tool_version` field in frontmatter (backward compatible)
- [ ] Supports directory scanning for batch import
