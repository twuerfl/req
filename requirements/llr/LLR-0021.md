## Description

The `req_lib` crate contains all domain data types for the `req` tool and nothing else. It has no I/O, no database access, no network calls, and no parsing dependencies beyond `serde` and `chrono`. It is the qualifiable kernel under ISO 26262 / DO-178C tool qualification.

## Implementation

- Location: `req_lib/src/`
- Entry point: `req_lib/src/lib.rs`
- Domain types: `req_lib/src/models.rs`
- `#![forbid(unsafe_code)]` declared in `lib.rs`
- `#![warn(missing_docs)]` declared in `lib.rs`
- Permitted dependencies: `serde` (with derive), `serde_json`, `chrono` (with serde feature)
- No other external crates permitted in `[dependencies]`

## Acceptance Criteria

- [ ] Crate compiles with `#![forbid(unsafe_code)]` — any unsafe block is a compile error
- [ ] All public types implement `Serialize` and `Deserialize`
- [ ] `Requirement`, `RequirementType`, `RequirementStatus`, `CodeRef`, `Link`, `LinkType`, `Coverage`, `AiExport`, `SCHEMA_VERSION` are all exported from the crate root
- [ ] `cargo clippy -p req_lib` reports zero warnings
- [ ] `cargo test -p req_lib` passes
