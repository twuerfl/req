## Description

When the `git` feature is enabled, the `req_engine` shall use `git2` to read `git blame`
data for each `CodeRef` span and determine the author identity for each line. It shall
report an independence violation when implementation (`REQ:`) and test (`VERIFIES:`) spans
of the same LLR share the same author identity.

## Author Identity Resolution (precedence order)

1. If a commit in the blame range contains the git trailer
   `Req-Agent-Session: <token>`, the token is used as the identity — two agent sessions
   with the same email but different tokens are treated as independent.
2. Otherwise, the git author email from the commit is used.

This allows separate AI agent runs (e.g., different Claude Code sessions) to be
recognised as independent even when they share the same committer email.

## Implementation Notes

- Module: `req_engine/src/audit/independence.rs`
- Feature-gated: only compiled when `features = ["git"]` (uses existing `git2` optional dep)
- Without the feature, the method returns `Ok(vec![])` and emits a single informational
  warning that the check was skipped
- Uncommitted changes: if `git2::Repository::status_file(path)` returns anything other
  than `Current`, that file is excluded from blame and an `IndependenceWarning` is added
  to the result (not an error)
- Entry point:
  `pub fn check_independence(base: &Path, code_refs: &[CodeRef], verifies_refs: &[CodeRef]) -> Result<IndependenceResult>`

## IndependenceResult Structure

```
violations:  Vec<IndependenceViolation>
warnings:    Vec<IndependenceWarning>
```

`IndependenceViolation`:
```
req_id:          String
impl_identities: Vec<String>
test_identities: Vec<String>
shared:          Vec<String>   (the overlapping identities)
```

`IndependenceWarning`:
```
req_id:  String
file:    PathBuf
reason:  String   (e.g., "uncommitted changes", "git blame unavailable")
```

## Acceptance Criteria

- [ ] Two spans with different author emails for the same LLR produce no violation
- [ ] Two spans with the same author email produce an `IndependenceViolation`
- [ ] Same email but different `Req-Agent-Session:` trailers produce no violation
- [ ] Without the `git` feature, the method returns `Ok` with empty results and one warning
- [ ] A file with uncommitted changes adds an `IndependenceWarning` (not error) and is
      excluded from the violation check
- [ ] Engine facade method: `pub fn audit_independence(&self) -> Result<IndependenceResult>`
