## Description

All export files (JSON, Markdown) shall include a schema version identifier to enable future migration and backward compatibility.

## Rationale

When the data model evolves, importers need to know which version of the schema they're dealing with. Without version identifiers, migration is guesswork and data corruption is likely.

## Acceptance Criteria

- [ ] JSON exports include `schema_version` field
- [ ] Markdown frontmatter includes `schema_version`
- [ ] Current schema version is `1.0.0`
- [ ] Import validates schema version and rejects unknown versions
- [ ] Import logs warning for older schema versions

## Implementation

```json
{
  "schema_version": "1.0.0",
  "exported_at": "2026-02-19T17:00:00Z",
  "tool_version": "0.1.0",
  "requirements": [...]
}
```

## Semantic Versioning

- **Major**: Breaking changes (fields removed, renamed)
- **Minor**: Additive changes (new optional fields)
- **Patch**: Bug fixes, no schema changes
