## Description

The `req_mcp` binary implements an MCP server using `rmcp` v1.2.0 with stdio transport. It registers all `req` engine operations as MCP tools and resources using the `#[tool]` and `#[tool_router]` macros, enabling AI models to invoke them via JSON-RPC over stdin/stdout.

## Implementation

- Location: `req_mcp/src/`
- Server handler: `req_mcp/src/server.rs` — implements `ServerHandler` trait via `#[tool_router]`
- Tool handlers: `req_mcp/src/tools.rs` — one method per MCP tool, annotated with `#[tool]`
- Resource handlers: `req_mcp/src/resources.rs` — MCP resource URIs
- Binary: `req_mcp/src/main.rs` — initialises `ReqEngine` and starts `rmcp` stdio server
- Transport: `rmcp::transport::stdio` (default)
- Each tool method accepts a JSON-deserializable input struct and returns a JSON-serializable output struct; JSON Schema is auto-generated by `rmcp-macros`

## MCP Tools

| Tool name | Engine method | Input | Output |
|-----------|--------------|-------|--------|
| `req.scan` | `ReqEngine::scan()` | `{ "dirs": [...] }` | `ScanResult` |
| `req.coverage` | `ReqEngine::coverage()` | `{}` | `Coverage` (traceability: HLR→LLR linkage, LLR implementation %) |
| `req.gaps` | `ReqEngine::gaps()` | `{}` | gap list |
| `req.check` | `ReqEngine::check()` | `{}` | issue list |
| `req.trace` | `ReqEngine::trace(id)` | `{ "id": "LLR-XXXX" }` | trace tree |
| `req.impact` | `ReqEngine::impact(id)` | `{ "id": "HLR-XXXX" }` | impact list |
| `req.list` | `ReqEngine::list()` | `{ "type": "hlr|llr|tst", "status": "..." }` | requirement list |
| `req.export` | `ReqEngine::export_ai()` | `{ "format": "json|ai-context" }` | export payload |
| `req_audit_coverage` | `ReqEngine::audit_coverage(report)` | `{ "report": "/path/to/coverage-report.json" }` | `Vec<LineCoverageScore>` (LLVM per-LLR line hit %) |

The `req_audit_coverage` tool parses a `cargo llvm-cov --json` report and returns per-LLR line-hit percentages. The `report` field is the path to the JSON file produced by `cargo llvm-cov --json --output-path <file>`. Returns an error if the file is absent or malformed.

## MCP Resources

In addition to the tools listed above, the following resources are exposed:

| URI | Engine call | Description |
| --- | --- | --- |
| `req://coverage` | `ReqEngine::coverage()` | Traceability coverage (same as `req.coverage` tool) |
| `req://gaps` | `ReqEngine::gaps()` | Traceability gaps |
| `req://requirements/{type}` | `ReqEngine::list_requirements(type)` | All requirements of a given type |
| `req://requirements/{type}/{id}` | list + filter by id | Single requirement |

## Acceptance Criteria

- [ ] `req-mcp-server` binary starts and responds to MCP `initialize` message
- [ ] All 9 MCP tools are registered and return correct JSON output when called
- [ ] All 4 MCP resources (`req://requirements/...`, `req://coverage`, `req://gaps`) are accessible
- [ ] `req_audit_coverage` returns `LineCoverageScore` array for a valid `cargo llvm-cov --json` report
- [ ] `req_audit_coverage` returns a tool error (not a server crash) when the report file is missing
- [ ] Server exits cleanly on stdin close
- [ ] `cargo test -p req_mcp` passes (including in-process transport tests)
