## Description

Tests for `req hooks install` and `req hooks uninstall`, verifying that the
hook file is created and removed correctly.

## Test Cases

### TC-001 — hooks install creates .git/hooks/pre-commit
Set up a TempDir with a `.git/hooks/` directory.
Call `crate::hooks::install_hook(false)` (non-strict mode).
Assert: `.git/hooks/pre-commit` exists and is non-empty.

### TC-002 — hooks uninstall removes the hook file
After install, call `crate::hooks::uninstall_hook()`.
Assert: `.git/hooks/pre-commit` no longer exists.

### TC-003 — hooks uninstall on absent file is safe (no panic or Err)
Call `uninstall_hook()` when no hook file exists.
Assert: result is `Ok`.

### TC-004 — install with strict=true writes strict flag into hook content
Call `install_hook(true)`. Read the file content.
Assert: file contains `--strict`.

## Acceptance Criteria

- [ ] TC-001 through TC-004 pass in the default build
- [ ] Tests located in `tests/cli_tests.rs`
- [ ] Each test uses `TempDir` for isolation; hook functions accept a base path
