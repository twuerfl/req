## Description

Tests for `req_engine/src/audit/coverage_map.rs`.

## Test Cases

### TC-001 — Parse valid LLVM coverage JSON and map to CodeRef

Provide a minimal valid LLVM coverage JSON with one file, segments covering lines 1–5
where lines 1–3 have `exec_count > 0` and lines 4–5 have `exec_count == 0`. Provide a
`CodeRef` covering lines 1–5.
Assert: `LineCoverageScore.lines_hit == 3`, `lines_total == 5`, `hit_percent == 60.0`.

### TC-002 — Line with exec_count == 0 is counted as missed

Segment at line 3 with `exec_count == 0` inside the CodeRef range.
Assert: that line is counted as missed.

### TC-003 — Lines outside all CodeRef ranges are ignored

Add segments for lines 50–60 where no CodeRef exists.
Assert: no `LineCoverageScore` entry for those lines.

### TC-004 — Malformed JSON returns Error::Parse

Pass a file containing invalid JSON.
Assert: result is `Err`.

### TC-005 — Path separators are normalised (Windows vs Unix)

Provide a JSON where `filename` uses backslashes; provide a `CodeRef` using forward
slashes to the same path. Assert: the match succeeds.

### TC-006 — Real `cargo llvm-cov` output is parsed correctly

Skip (not fail) when `cargo-llvm-cov` is not installed or `llvm-tools-preview`
is absent. When present:

1. Run `cargo llvm-cov --manifest-path tests/fixtures/coverage_fixture/Cargo.toml
   --json --output-path <tmp>` against the fixture crate.
2. Build two `CodeRef`s: one for `add_numbers` (covered by `#[test]`), one for
   `never_called` (never invoked).
3. Register both in a fresh engine cache with parent LLR requirements.
4. Call `engine.audit_coverage(<tmp>)`.

Assert: covered function has `lines_hit > 0`; uncovered function has `lines_hit == 0`.

## Acceptance Criteria

- [ ] TC-001 through TC-005 pass in the default build
- [ ] TC-006 passes when `cargo-llvm-cov` is installed; skips gracefully otherwise
- [ ] Tests use `tempfile` for JSON fixture files
- [ ] The fixture crate `tests/fixtures/coverage_fixture` is excluded from the workspace
