## Description

Integration tests for `.req/config.toml` loading, verifying that custom
`source_dirs` and `requirements_path` are picked up by `ReqEngine::open`.

## Test Cases

### TC-001 — Default config is used when .req/config.toml is absent
Init a project without a config file. Call `ReqEngine::open`.
Assert: engine opens without error and uses default source dirs.

### TC-002 — Custom source_dirs in config are respected
Write `.req/config.toml` with `source_dirs = ["custom_src"]`.
Create `custom_src/` directory. Call `engine.scan(None, false)`.
Assert: scan completes without error and reports the custom directory.

### TC-003 — Missing config file does not panic
Delete `.req/config.toml` after init. Call `ReqEngine::open`.
Assert: result is `Ok` (falls back to defaults).

### TC-004 — Invalid TOML in config returns clear Err
Write `.req/config.toml` with invalid TOML content.
Assert: `ReqEngine::open` returns `Err` with a descriptive message.

## Acceptance Criteria

- [ ] TC-001 through TC-004 pass in the default build
- [ ] Tests located in `tests/config_tests.rs`
- [ ] Each test uses `TempDir` for isolation
