## Description

Integration tests for the `JsonAdapter`: export to JSON then re-import, verifying
that all fields survive the roundtrip without loss or mutation.

## Test Cases

### TC-001 — Export then import preserves all core fields
Write a `Requirement` with all fields set (id, type, title, text, status, parent,
aliases, attributes). Export via `engine.export("json", None)`. Write to a temp file.
Import via `engine.import(path, "json", None)`.
Assert: imported requirement has identical id, title, text, status, parent.

### TC-002 — Import from JSON array adds requirements to the store
Init a project. Write a JSON file containing two requirements. Call
`engine.import(path, "json", None)`.
Assert: `engine.list_requirements(None)` returns the two new requirements.

### TC-003 — JSON export includes all requirements
Import three requirements of different types. Call `engine.export("json", None)`.
Assert: parsed JSON array length equals total stored requirements.

### TC-004 — Import of malformed JSON returns Err
Write a file containing `{broken json`.
Assert: `engine.import(path, "json", None)` returns `Err`.

## Acceptance Criteria

- [ ] TC-001 through TC-004 pass in the default build
- [ ] Tests located in `tests/export_tests.rs`
- [ ] Each test uses `TempDir` for isolation
