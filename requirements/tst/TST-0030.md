## Description

Compile-time and runtime checks that `req_lib` meets its architecture constraints:
no unsafe code, correct public exports, all types serialisable.

## Test Cases

### TC-001 — All core types are accessible from req_lib crate root
In a test (or by compilation), access:
`req_lib::Requirement`, `req_lib::RequirementType`, `req_lib::RequirementStatus`,
`req_lib::CodeRef`, `req_lib::Link`, `req_lib::LinkType`, `req_lib::Coverage`,
`req_lib::AiExport`, `req_lib::SCHEMA_VERSION`.
Assert: all names resolve (compilation is the test).

### TC-002 — Requirement implements Serialize and Deserialize
Construct a `Requirement` and roundtrip via `serde_json`.
Assert: deserialized value equals original.

### TC-003 — Coverage implements Serialize and Deserialize
Construct a `Coverage` and roundtrip via `serde_json`. Assert: values equal.

### TC-004 — SCHEMA_VERSION is a non-empty string
Assert: `req_lib::SCHEMA_VERSION.len() > 0`.

## Acceptance Criteria

- [ ] TC-001 through TC-004 pass under `cargo test -p req_lib`
- [ ] Tests located in `tests/arch_tests.rs`
