## Description

Integration tests for `engine.coverage()`, verifying correct HLR and LLR
coverage percentages for known fixture data.

## Test Cases

### TC-001 — Full chain: HLR with LLR and code ref gives 100% coverage
Seed one HLR and one LLR (child of HLR) plus one code ref (VERIFIES the LLR).
Call `engine.coverage()`.
Assert: `hlr_with_llr == 1`, `llr_implemented == 1`, `llr_tested == 1`.

### TC-002 — HLR without LLR shows 0% HLR coverage
Seed one HLR only. Call `engine.coverage()`.
Assert: `hlr_total == 1`, `hlr_with_llr == 0`.

### TC-003 — LLR without code ref shows not implemented
Seed one HLR and one LLR with no code refs. Call `engine.coverage()`.
Assert: `llr_implemented == 0`, `llr_tested == 0`.

### TC-004 — LLR with REQ tag but no VERIFIES tag shows implemented but not tested
Seed one LLR and insert a `code_refs` row (type = "req", not "verifies").
Assert: `llr_implemented == 1`, `llr_tested == 0`.

### TC-005 — Coverage percentages are correct fractions
Seed 4 LLRs, 2 implemented, 1 tested. Call `engine.coverage()`.
Assert: `llr_implementation_pct()` ≈ 50.0 and `llr_test_pct()` ≈ 25.0.

## Acceptance Criteria

- [ ] TC-001 through TC-005 pass in the default build
- [ ] Tests located in `tests/coverage_report_tests.rs`
- [ ] Each test uses `TempDir` for isolation
