## Description

Integration tests for the `req remove` command covering normal removal, cache-only mode,
error conditions, and dependency guard behaviour.

## Test Cases

### TC-0036-01 ā€” Normal removal deletes file and cache entries

**Given** a repository with requirement `LLR-0099` whose `.md` file exists on disk and
whose ID is present in the `requirements`, `file_hashes`, and `links` cache tables
**When** `req remove LLR-0099 --yes` is executed
**Then** the `.md` file no longer exists on disk, the `requirements` row for `LLR-0099`
is absent from the cache, all `code_refs` rows referencing `LLR-0099` are absent, and
the command exits with code 0

### TC-0036-02 ā€” `--cache-only` leaves file on disk

**Given** the same setup as TC-0036-01
**When** `req remove LLR-0099 --cache-only --yes` is executed
**Then** the `.md` file still exists on disk and the cache rows for `LLR-0099` are absent

### TC-0036-03 ā€” Non-existent ID exits non-zero

**Given** an ID that does not exist in the cache or on disk
**When** `req remove LLR-9999` is executed
**Then** the command exits non-zero and stderr contains an actionable error message

### TC-0036-04 ā€” Dependent child triggers warning

**Given** requirement `HLR-0099` that is the `parent` of `LLR-0099`
**When** `req remove HLR-0099 --yes` is executed (without `--strict` or `--force`)
**Then** the command prints a warning to stderr mentioning the dependent LLR, but
completes successfully and exits 0

### TC-0036-05 ā€” `--strict` with dependents aborts removal

**Given** the same setup as TC-0036-04
**When** `req remove HLR-0099 --yes --strict` is executed
**Then** the command exits non-zero, stderr describes the blocking dependents, and
neither the file nor the cache is modified

### TC-0036-06 ā€” `--force` bypasses dependency check

**Given** the same setup as TC-0036-04
**When** `req remove HLR-0099 --yes --force` is executed
**Then** the file is deleted and cache entries are purged despite the existing dependent

### TC-0036-07 ā€” `import_sources` entry is also purged

**Given** a requirement that was imported and has a row in the `import_sources` table
**When** `req remove <ID> --yes` is executed
**Then** the `import_sources` row for that ID is absent from the cache after removal

### TC-0036-08 ā€” Confirmation prompt without `--yes`

**Given** a valid requirement ID
**When** `req remove <ID>` is executed interactively and the user inputs `n` at the
confirmation prompt
**Then** the command exits 0 without modifying the file or cache
