## Description

Tests that `req_cli` meets its architecture constraints: binary named `req`,
no business logic in the crate, clean build.

## Test Cases

### TC-001 — Binary artifact is named "req"
Assert: `target/debug/req.exe` (or `req` on Unix) exists after `cargo build -p req_cli`.
(Verified by the [[bin]] name = "req" entry in req_cli/Cargo.toml.)

### TC-002 — req_cli/Cargo.toml declares [[bin]] name = "req"
Read `req_cli/Cargo.toml` and assert it contains `name = "req"` under `[[bin]]`.

### TC-003 — req_cli does not depend on req_engine internals directly
Assert: `req_cli/Cargo.toml` lists only `req_engine` (not `req_lib` or internal modules)
as a path dependency.

## Acceptance Criteria

- [ ] TC-001 through TC-003 pass under `cargo test`
- [ ] Tests located in `tests/arch_tests.rs`
