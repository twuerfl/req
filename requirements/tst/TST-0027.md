## Description

Tests that verify the CLI command surface — all commands are present in `req --help`,
key flags exist, and the binary is named `req`.

## Test Cases

### TC-001 — req --help lists all 19 commands
Invoke `req --help` (via process spawn or by inspecting clap metadata).
Assert: output contains each command name: init, new, scan, trace, coverage, gaps,
impact, check, export, import, import-reqif, export-reqif, list, hooks, ci,
import-ai, check-provenance, migrate, audit.

### TC-002 — Global --base and --format flags are present
Assert: `req --help` output contains `--base` and `--format`.

### TC-003 — Global --strict flag is present
Assert: `req --help` output contains `--strict`.

### TC-004 — req audit --help lists all 6 subcommands
Assert: output contains triviality, criteria, mutation, coverage, export-context, independence.

### TC-005 — Binary name is "req"
Assert: the `req --help` usage line starts with `req` (not `req_cli`).

## Acceptance Criteria

- [ ] TC-001 through TC-005 pass in the default build
- [ ] Tests located in `tests/cli_tests.rs`
- [ ] Tests use process spawn via `std::process::Command`
