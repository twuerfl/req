## Description

Tests for `req_engine/src/audit/triviality.rs` and the `ReqEngine::audit_triviality`
facade method.

## Test Cases

### TC-001 — Tautological assertion detected (P1, Error)
Create a temp file containing:
```rust
// VERIFIES: LLR-0001
#[test]
fn t() { assert!(true); }
```
Call `analyse_code_ref(path, 1, Some(3))`.
Assert: result contains one finding with `pattern == TautologicalAssertion` and
`severity == Error`.

### TC-002 — Zero substantive lines detected (P3, Error)
Create a temp file containing only a tag comment and blank lines.
Assert: result contains a finding with `pattern == ZeroSubstantiveLines` and `severity == Error`.

### TC-003 — Legitimate function produces no findings
Create a temp file with a multi-line function containing real logic (variable binding,
conditionals, function calls). Assert: result is empty.

### TC-004 — Stub body detected (P2, Warning)
Create a temp file with `fn foo() -> Result<()> { Ok(()) }` tagged with `// REQ: LLR-0001`.
Assert: result contains one finding with `pattern == StubBody` and `severity == Warning`.

### TC-005 — Files in `examples/` are skipped
Create a temp file under a path containing `examples/` as a path component.
Assert: `analyse_code_ref` returns an empty result (no findings).

### TC-006 — `#[allow(dead_code)]` downgrades P2 to Info
Create a temp file with `#[allow(dead_code)]` on the function and `fn foo() { Ok(()) }`.
Assert: finding severity is `Info`, not `Warning`.

### TC-007 — `audit_triviality()` aggregates findings from all CodeRefs
Set up a `TempDir` with `ReqEngine::init`, seed two `CodeRef`s via the cache
(one trivial, one legitimate). Call `engine.audit_triviality()`.
Assert: exactly one `TrivialityReport` with one finding; the legitimate span is absent.

## Acceptance Criteria

- [ ] TC-001 through TC-007 pass in the default build (`cargo test`)
- [ ] No external tools or network access required
