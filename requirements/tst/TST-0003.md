## Description

Test suite verifying the Markdown adapter for reading/writing requirements.

## Test Cases

### TC-001: Parse Markdown with Frontmatter
Verify parsing of markdown files with YAML frontmatter:
- Extract id, type, title, status, parent
- Handle optional fields (aliases, attributes)

### TC-002: Parse Body Text
Verify body text extraction:
- Remove title heading from body
- Trim whitespace
- Preserve formatting

### TC-003: Write Requirement to File
Verify writing requirements to markdown:
- Generate valid YAML frontmatter
- Write body text correctly
- Create parent directories if needed

### TC-004: Round-trip Parse/Write
Verify that parse → write → parse preserves data:
- All fields preserved
- No data loss

### TC-005: Edge Cases

Verify robustness against malformed input:

- Missing frontmatter handled gracefully
- Invalid YAML returns a parse error (not a panic)
- Empty body text supported

### TC-006: tool_version in Frontmatter

Verify the writer includes `tool_version` in YAML frontmatter (implements LLR-0013):

- Written file contains `tool_version:` field
- Value matches the current `CARGO_PKG_VERSION`
- Parser tolerates `tool_version` field without error
- Round-trip parse/write preserves `tool_version` without data loss
