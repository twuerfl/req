## Description

Test suite verifying AI suggestion import with safety guarantees.

## Test Cases

### TC-001: Draft Status Enforcement
Verify all imported suggestions are draft:
- AI suggestions cannot be auto-approved
- Status is always set to "draft"

### TC-002: ID Validation
Verify ID validation on import:
- Valid IDs accepted
- Invalid IDs rejected with error

### TC-003: Parent Validation for LLR
Verify LLR parent requirement:
- LLR without parent rejected
- Parent must exist in cache

### TC-004: Duplicate Detection
Verify duplicate ID detection:
- Existing IDs rejected
- New IDs generated if not provided

### TC-005: Interactive Mode
Verify interactive import:
- User can accept/reject suggestions
- Abort option works

### TC-006: Provenance Tracking

Verify provenance metadata is recorded (LLR-0013):

- Each change records author, timestamp, and source (human/ai)
- Protected operations (approve, delete) enforce provenance check
- Provenance chain is queryable
