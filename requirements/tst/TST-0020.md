## Description

Integration tests for `req export --format json` and `req export --format markdown`,
exercising the engine directly via `ReqEngine` in a `TempDir` fixture.

## Test Cases

### TC-001 — JSON export returns valid JSON array
Init a project, import two requirements. Call `engine.export("json", None)`.
Assert: output parses as a JSON array with length ≥ 2.

### TC-002 — JSON export with --id returns single requirement
Call `engine.export("json", Some("HLR-0001"))`.
Assert: output parses as a JSON array of exactly one element with `id == "HLR-0001"`.

### TC-003 — JSON export unknown ID returns Err
Call `engine.export("json", Some("HLR-9999"))`.
Assert: result is `Err`.

### TC-004 — Markdown export contains YAML frontmatter
Call `engine.export("markdown", None)`.
Assert: output contains `---` delimiter and `id:` field.

### TC-005 — Markdown export with --id returns single block
Call `engine.export("markdown", Some("HLR-0001"))`.
Assert: output contains exactly one `id: HLR-0001` occurrence.

### TC-006 — Unknown format returns Err
Call `engine.export("xml", None)`.
Assert: result is `Err` containing "Unknown export format".

## Acceptance Criteria

- [ ] TC-001 through TC-006 pass in the default build
- [ ] Tests located in `tests/export_tests.rs`
- [ ] Each test uses `TempDir` for isolation
