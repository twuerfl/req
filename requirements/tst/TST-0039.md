## Description

Tests verifying that all `req` commands producing structured output are deterministic:
identical repository state must yield byte-identical JSON output across repeated
invocations, independent of execution order, platform, or environment.

## Test Cases

### TC-0039-01 — `req list --format json` is sorted by requirement ID

**Given** a repository containing requirements of mixed types and IDs in arbitrary
insertion order
**When** `req list --format json` is executed twice on the same repository state
**Then** both outputs are byte-identical and the `id` fields in the JSON array appear in
lexicographic ascending order (e.g. HLR-0001 before HLR-0002, LLR-0001 before LLR-0002)

### TC-0039-02 — `req coverage --format json` contains no wall-clock timestamps or PIDs

**Given** a repository with at least one covered LLR
**When** `req coverage --format json` is executed at two different wall-clock times
**Then** both outputs are byte-identical; the JSON object contains no field whose value
changes between runs (no `generated_at`, `timestamp`, `pid`, or machine-specific data)

### TC-0039-03 — `req gaps --format json` output list is sorted

**Given** a repository with multiple traceability gaps
**When** `req gaps --format json` is executed twice
**Then** both outputs are byte-identical and every array within the JSON object
(e.g. `llr_without_code`, `undefined_ids`) is in lexicographic ascending order

### TC-0039-04 — `req check --format json` output list is sorted

**Given** a repository with multiple validation warnings or errors
**When** `req check --format json` is executed twice
**Then** both outputs are byte-identical and the issues array is sorted by
`requirement_id` in lexicographic ascending order

### TC-0039-05 — File paths in JSON output use forward slashes and are relative to `--base`

**Given** a repository on a Windows host (where native paths use backslashes) and a
`--base` directory that is a parent of all source files
**When** any command that includes file paths in its `--format json` output is executed
(e.g. `req coverage`, `req gaps`, `req audit`)
**Then** all path strings in the JSON contain only forward slashes (`/`) and begin with
a path segment relative to `--base` — no drive letters, no UNC prefixes, no backslashes

### TC-0039-06 — `req export --format json` is identical for identical repository state

**Given** a repository at a fixed commit (no uncommitted changes)
**When** `req export --format json` is executed twice with the same arguments
**Then** the two outputs are byte-identical; in particular the exported JSON contains no
`generated_at` or equivalent timestamp field in the requirements array or its envelope
