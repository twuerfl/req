## Description

Test suite verifying the Requirement data model implementation.

## Test Cases

### TC-001: Requirement Struct Fields
Verify that Requirement struct contains all required fields:
- id: String
- req_type: RequirementType (HLR/LLR/TST)
- title: String
- text: String
- status: RequirementStatus
- parent: Option<String>
- aliases: Vec<String>
- attributes: HashMap<String, String>

### TC-002: ID Format Validation
Verify ID format validation:
- Valid: HLR-0001, LLR-0123, TST-0042
- Invalid: invalid, XYZ-001, HLR-ABC

### TC-003: Auto-generation of IDs
Verify sequential ID generation:
- First HLR should be HLR-0001
- Next HLR should increment number
- Different types have separate sequences
