## Description

Integration tests for the `import_sources` tracking table and the incremental scan
staleness detection logic introduced by LLR-0037.

## Test Cases

### TC-0038-01 ŌĆö Import populates `import_sources`

**Given** a valid JSON import file at a known path
**When** `req import <file>` is executed
**Then** the `import_sources` table contains one row per imported requirement with the
correct `source_path`, a non-empty `sha256`, and a non-empty `imported_at` timestamp

### TC-0038-02 ŌĆö Unchanged source remains clean after scan

**Given** requirements imported from a source file that has not changed since import
**When** `req scan` is executed
**Then** `import_status` for those requirements is NULL and `req check` exits 0

### TC-0038-03 ŌĆö Modified source triggers `import_stale`

**Given** requirements imported from a source file
**When** the source file is modified (altering its SHA-256) and `req scan` is run
**Then** `import_status` for the affected requirements is `stale` and `req check`
reports them as errors and exits non-zero

### TC-0038-04 ŌĆö Deleted source triggers `import_orphaned`

**Given** requirements imported from a source file
**When** the source file is deleted from disk and `req scan` is run
**Then** `import_status` for the affected requirements is `orphaned` and `req check`
reports them as errors and exits non-zero

### TC-0038-05 ŌĆö Re-import clears stale flag

**Given** requirements with `import_status = 'stale'` (source was modified)
**When** `req import <updated-source-file>` is re-run with the same output path
**Then** `import_sources.sha256` is updated to the new hash, `import_status` is NULL,
and `req check` exits 0

### TC-0038-06 ŌĆö `req remove` deletes `import_sources` row

**Given** a requirement with a row in `import_sources`
**When** `req remove <ID> --yes` is executed
**Then** the `import_sources` row for that ID is absent

### TC-0038-07 ŌĆö `req list` shows `import_status` when flags present

**Given** at least one requirement with `import_status = 'orphaned'`
**When** `req list` is executed
**Then** the output includes an `import_status` column showing `orphaned` for that
requirement

### TC-0038-08 ŌĆö `req gaps` surfaces orphaned imports

**Given** requirements with `import_status = 'orphaned'`
**When** `req gaps` is executed
**Then** the output includes the orphaned requirements as unresolved gaps

### TC-0038-09 ŌĆö `req import-ai` and `req import-reqif` also populate table

**Given** import via `req import-ai` and `req import-reqif` respectively
**When** each command completes
**Then** each imported requirement has a corresponding row in `import_sources`

### TC-0038-10 ŌĆö `--source` override updates stored path without re-import

**Given** a requirement with an existing `import_sources` row
**When** `req import --source <new-path>` is executed with the corrected path
**Then** `import_sources.source_path` is updated to `new-path` and the SHA-256 is
recomputed from that file
