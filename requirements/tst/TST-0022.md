## Description

Integration tests for `engine.migrate()` and backward-compatibility import of
legacy requirement files missing optional fields.

## Test Cases

### TC-001 — migrate dry_run reports files without writing
Write a requirement file with non-canonical formatting. Call `engine.migrate(true)`.
Assert: `result.migrated` is non-empty and the file on disk is unchanged.

### TC-002 — migrate rewrites non-canonical file
Call `engine.migrate(false)`.
Assert: `result.migrated` is non-empty and the rewritten file parses correctly.

### TC-003 — migrate is idempotent
Call `engine.migrate(false)` twice. Assert: second call returns `result.migrated` empty.

### TC-004 — Import accepts file without optional fields
Write a minimal Markdown requirement with only `id`, `type`, `title`, `status`.
Parse via `MarkdownAdapter`. Assert: result is `Ok` and `parent` is `None`.

### TC-005 — Unknown YAML fields are preserved in attributes
Write a requirement with an extra `custom_field: foo` in frontmatter.
Assert: parsed requirement has `attributes["custom_field"] == "foo"`.

## Acceptance Criteria

- [ ] TC-001 through TC-005 pass in the default build
- [ ] Tests located in `tests/migrate_tests.rs`
- [ ] Each test uses `TempDir` for isolation
