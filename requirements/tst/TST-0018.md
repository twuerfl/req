## Description

Tests for `req_engine/src/audit/independence.rs` and the `ReqEngine::audit_independence`
facade method.

Note: TC-001 through TC-003 and TC-005 require the `git` feature and a real Git repository
fixture. TC-004 runs in the default build.

## Test Cases

### TC-001 — Different author emails → no violation  (feature: git)
Initialise a temp Git repository. Commit a source file with a `REQ: LLR-XXXX` tag using
author `agent-a@example.com`. Commit a test file with `VERIFIES: LLR-XXXX` using author
`agent-b@example.com`. Call `engine.audit_independence()`.
Assert: `IndependenceResult.violations` is empty.

### TC-002 — Same author email → violation reported  (feature: git)
Repeat TC-001 but use the same author email for both commits.
Assert: `violations` contains one `IndependenceViolation` for the LLR, with
`shared` containing the common email.

### TC-003 — Different Req-Agent-Session trailers → no violation  (feature: git)
Use the same author email but add `Req-Agent-Session: session-A` to the implementation
commit and `Req-Agent-Session: session-B` to the test commit.
Assert: `violations` is empty (different sessions = independent).

### TC-004 — Without git feature, returns Ok with empty violations
Call `check_independence` compiled without the `git` feature.
Assert: returns `Ok(IndependenceResult { violations: vec![], warnings: vec![...] })` where
`warnings` contains one entry indicating the check was skipped.

### TC-005 — Uncommitted file produces IndependenceWarning, not error  (feature: git)
Set up a repo with one committed implementation file and one *unstaged* test file.
Assert: `audit_independence()` returns `Ok`; `warnings` contains one entry mentioning
the uncommitted file; `violations` is empty.

## Acceptance Criteria

- [ ] TC-004 passes in the default build (`cargo test`)
- [ ] TC-001, TC-002, TC-003, TC-005 pass when built with `--features git`
- [ ] No violation is ever returned as an `Err` — violations are data, not errors
