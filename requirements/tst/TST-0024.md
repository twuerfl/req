## Description

Integration tests for the `attributes` field on `Requirement`, verifying
serialization to/from Markdown frontmatter and JSON.

## Test Cases

### TC-001 — Attributes roundtrip through Markdown
Create a `Requirement` with `attributes = {"risk": "high", "source": "DOORS-42"}`.
Serialize via `MarkdownAdapter::format_to_string`. Re-parse via `MarkdownAdapter::parse_content`.
Assert: parsed `attributes` map contains both key-value pairs unchanged.

### TC-002 — Attributes roundtrip through JSON
Serialize the same requirement via `serde_json::to_string`. Deserialize back.
Assert: attributes are identical.

### TC-003 — Requirement without attributes section parses cleanly
Write a Markdown file with no `attributes:` key in frontmatter.
Assert: parsed `attributes` map is empty (not an error).

### TC-004 — Attribute keys are preserved exactly (case-sensitive)
Set `attributes = {"CamelCase": "v1", "snake_case": "v2"}`.
Roundtrip via Markdown. Assert: both keys survive unchanged.

## Acceptance Criteria

- [ ] TC-001 through TC-004 pass in the default build
- [ ] Tests located in `tests/export_tests.rs`
- [ ] Each test uses isolated `Requirement` values (no TempDir required)
