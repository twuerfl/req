## Description

Tests that `req check` returns exit code 0 on a clean project and exit code 1
when `--strict` is set and there are validation failures.

## Test Cases

### TC-001 — req check exits 0 on a clean project
Init a project with fully linked requirements and valid code refs.
Call `engine.validate()`. Assert: result is `Ok` with empty issues vec.

### TC-002 — req check exits 1 in strict mode with broken link
Insert a code ref pointing to a non-existent requirement ID.
Call `engine.validate()`. Assert: result contains at least one `ValidationIssue`.
(CLI would exit 1 with --strict based on this condition.)

### TC-003 — req check exits 0 with warnings but without --strict
Insert a warning-level issue. Assert: non-strict mode returns `Ok` (exit 0).

## Acceptance Criteria

- [ ] TC-001 through TC-003 pass in the default build
- [ ] Tests located in `tests/cli_tests.rs`
- [ ] Each test uses `TempDir` for isolation
