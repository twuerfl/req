## Description

End-to-end integration tests for the `req audit` subcommand introduced by LLR-0032.
Tests drive the `ReqEngine` audit methods directly (not via process spawn) to avoid
coupling tests to the binary interface.

## Test Cases

### TC-001 — audit_triviality on clean project exits with empty results
Set up a `TempDir` with `ReqEngine::init`, seed a legitimate (non-trivial) CodeRef.
Call `engine.audit_triviality()`.
Assert: result is `Ok` and the returned `Vec<TrivialityReport>` is empty.

### TC-002 — audit_triviality detects seeded trivial span
Seed a CodeRef pointing to a file containing `assert!(true)`.
Call `engine.audit_triviality()`.
Assert: result contains at least one `TrivialityReport` with `Error` severity.

### TC-003 — audit_criteria reports gap for LLR with unlinked criterion
Seed an LLR file with two `- [ ]` items; seed only one `criterion_links` row (for `#1`).
Call `engine.audit_criteria()`.
Assert: the returned `CriteriaReport` has `linked: false` for the second criterion.

### TC-004 — audit_mutation with malformed JSON returns Err
Call `engine.audit_mutation(path_to_invalid_json)`.
Assert: result is `Err`.

### TC-005 — audit_export produces AuditBundle with correct LLR id
Seed an LLR "LLR-TEST-01" and a CodeRef for it. Call `engine.audit_export("LLR-TEST-01", None, None)`.
Assert: `bundle.llr.id == "LLR-TEST-01"` and `bundle.schema_version == "2.0.0"`.

### TC-006 — audit_triviality with Error findings triggers strict mode
Call `engine.audit_triviality()` on a project with a seeded trivial span.
Collect findings. Assert: at least one finding has `severity == Error` (CLI would exit 1
in strict mode based on this condition).

## Acceptance Criteria

- [ ] TC-001 through TC-006 pass in the default build
- [ ] Tests are located in `tests/audit_tests.rs`
- [ ] Each test uses `TempDir` for isolation
