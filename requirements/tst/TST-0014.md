## Description

Tests for `req_engine/src/audit/mutation.rs`.

## Test Cases

### TC-001 — Parse valid mutants JSON and count scores
Provide a minimal valid JSON array with two mutants for the same file/range — one
`"Caught"` and one `"Missed"`. Provide a matching `CodeRef` covering those lines.
Assert: `MutationScore.caught == 1`, `missed == 1`, `mutants_total == 2`,
`score_percent == 50.0`.

### TC-002 — Mutant overlapping CodeRef range is attributed correctly
Mutant at line 10; `CodeRef` covers lines 8–15. Assert: attributed to the CodeRef's `req_id`.

### TC-003 — Mutant outside all CodeRef ranges goes to untagged count
Provide a mutant at line 100; no CodeRef covers line 100.
Assert: `untagged_mutants == 1` in the returned summary; no `MutationScore` entries.

### TC-004 — Malformed JSON returns Error::Parse
Pass a file containing `{ not valid json }`.
Assert: result is `Err` and the error message contains "parse" or "json".

### TC-005 — Unviable mutants count in total but not caught/missed
Provide one `"Unviable"` mutant overlapping a CodeRef.
Assert: `mutants_total == 1`, `caught == 0`, `missed == 0`.

## Acceptance Criteria

- [ ] TC-001 through TC-005 pass in the default build
- [ ] Tests use `tempfile` for JSON fixture files, not hardcoded paths
