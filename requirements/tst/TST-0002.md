## Description

Test suite verifying the code scanner implementation for inline requirement tags.

## Test Cases

### TC-001: Single-line REQ Tags
Verify parsing of single-line requirement tags:
- `// REQ: LLR-0001` - single requirement
- `/* REQ: LLR-0002, LLR-0003 */` - multiple requirements
- `// VERIFIES: TST-0042` - verification tags

### TC-002: Requirement ID Validation
Verify ID validation function:
- Valid IDs: HLR-0001, LLR-0123, TST-0042
- Invalid IDs: invalid, XYZ-001

### TC-003: Block Tags (BEGIN-REQ/END-REQ)
Verify multi-line block tags:
- BEGIN-REQ marks start of block
- END-REQ marks end of block
- line_end is set correctly for blocks

### TC-004: Mixed Tags
Verify single-line and block tags work together:
- Single-line tags have line_end = None
- Block tags have line_end = Some(n)

### TC-005: Unclosed Block Handling
Verify graceful handling of unclosed blocks:
- Treated as single-line tag
- No crash or panic

### TC-006: Orphan END-REQ Handling
Verify orphan END-REQ tags are ignored:
- No matching BEGIN-REQ
- No tag created
