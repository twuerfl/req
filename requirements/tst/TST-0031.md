## Description

Tests that `req_engine` meets its architecture constraints: all operations
exposed via `ReqEngine`, req_lib types re-exported, no presentation logic.

## Test Cases

### TC-001 — All req_lib public types accessible via req_engine
Access `req_engine::Requirement`, `req_engine::Coverage`, `req_engine::Link`,
`req_engine::CodeRef`, `req_engine::SCHEMA_VERSION`.
Assert: all names resolve (compilation is the test).

### TC-002 — ReqEngine exposes scan, coverage, gaps, validate, export, import
Verify by calling each method on a freshly initialised engine in a TempDir.
Assert: each call returns `Ok` (or expected Err for invalid input).

### TC-003 — req_engine has no `colored` or terminal crate dependency
Assert: `cargo metadata -p req_engine` does not list `colored` or `indicatif`
in the direct dependencies. (Checked by reading Cargo.toml in the test.)

## Acceptance Criteria

- [ ] TC-001 through TC-003 pass under `cargo test`
- [ ] Tests located in `tests/arch_tests.rs`
