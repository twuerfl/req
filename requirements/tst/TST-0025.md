## Description

Tests that `SCHEMA_VERSION` is present in exports and that the version constant
matches the expected value.

## Test Cases

### TC-001 — SCHEMA_VERSION constant equals "1.0.0"
Assert at compile/test time: `req_lib::SCHEMA_VERSION == "1.0.0"`.

### TC-002 — AiExport struct includes schema_version field set to SCHEMA_VERSION
Construct `AiExport::new(vec![], vec![], vec![], coverage)`.
Assert: `export.schema_version == req_lib::SCHEMA_VERSION`.

### TC-003 — AiExport serialises schema_version into JSON
Serialize to JSON. Assert: the JSON string contains `"schema_version":"1.0.0"`.

### TC-004 — AiExport includes tool_version field
Assert: `export.tool_version` is non-empty.

## Acceptance Criteria

- [ ] TC-001 through TC-004 pass in the default build
- [ ] Tests located in `tests/export_tests.rs`
