## Description

Tests for `ReqEngine::audit_export` and the `AuditBundle` data type.

## Test Cases

### TC-001 — Bundle produced without optional tool reports
Set up a `TempDir` with `ReqEngine::init`, seed one LLR and one `CodeRef`.
Call `engine.audit_export("LLR-XXXX", None, None)`.
Assert: `mutation_score` is `None`, `line_coverage_score` is `None`, `implementation_spans`
is non-empty, `prompt_hint` is non-empty.

### TC-002 — Bundle serialises and deserialises without data loss
Call `audit_export`, serialise the result with `serde_json::to_string_pretty`, then
deserialise back with `serde_json::from_str::<AuditBundle>`.
Assert: all fields are equal to the original (round-trip).

### TC-003 — source_text contains only span lines, not entire file
Create a source file with 50 lines; tag lines 10–15.
Assert: `SourceSpan.source_text` in `implementation_spans` contains lines 10–15 and
does NOT contain line 1 or line 50.

### TC-004 — prompt_hint references all six review tasks
Assert: `bundle.prompt_hint` contains the words "acceptance_criteria", "test_spans",
"criterion", "triviality", "unlinked", and "verdict".

### TC-005 — Unreadable source file adds warning rather than returning error
Seed a `CodeRef` pointing to a file that does not exist.
Assert: `audit_export` returns `Ok`, the `warnings` field (or equivalent) is non-empty,
and `implementation_spans` for that ref contains an empty `source_text` or a warning note.

### TC-006 — schema_version is "2.0.0" and generated_at is valid ISO 8601
Assert: `bundle.schema_version == "2.0.0"` and `bundle.generated_at` is parseable as an
ISO 8601 datetime.

## Acceptance Criteria

- [ ] TC-001 through TC-006 pass in the default build
- [ ] No network calls are made during any test
