## Description

Feature-gated tests for ReqIF import and export. Only runs when compiled with
`--features reqif`. The existing skeleton in `tests/reqif_tests.rs` covers
basic adapter wiring; these cases cover the full roundtrip.

## Test Cases

### TC-001 — Import .reqif file yields correctly typed requirements
(Feature-gated) Read a fixture `.reqif` file. Call `ReqifAdapter::read`.
Assert: result is `Ok` and requirements have correct `req_type` values.

### TC-002 — Export to .reqif then re-import is a lossless roundtrip
(Feature-gated) Export a vec of requirements to a temp `.reqif` file.
Re-import. Assert: IDs and titles match.

### TC-003 — can_handle rejects non-.reqif files (non-feature-gated)
Call `ReqifAdapter::can_handle` on a `.md` file.
Assert: returns `false`. (Already covered in `reqif_tests.rs`.)

### TC-004 — read without reqif feature returns Config error (non-feature-gated)
Call `ReqifAdapter::read` without the feature enabled.
Assert: returns `Err(Error::Config(...))`.

## Acceptance Criteria

- [ ] TC-003 and TC-004 pass in the default build
- [ ] TC-001 and TC-002 pass under `cargo test --features reqif`
- [ ] Tests located in `tests/reqif_tests.rs`
