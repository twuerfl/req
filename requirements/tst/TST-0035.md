## Description

Minimal tests for LLR-0123, the documentation placeholder requirement.
Verifies that the placeholder ID is well-formed and recognised by the
requirement model, so that scanner doc-comment references to `LLR-0123`
never appear as orphan code refs.

## Test Cases

### TC-001 — LLR-0123 ID parses as a valid LLR identifier
Assert: `Requirement::parse_id("LLR-0123")` returns `Some((RequirementType::Llr, 123))`.

### TC-002 — Placeholder requirement can be stored and retrieved
Seed `LLR-0123` into a fresh engine cache. Assert: `engine.validate()` does
not produce an `Error`-severity issue for that ID.
