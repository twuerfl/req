## Description

Tests for the ReqIF adapter (`req_engine/src/adapter/reqif.rs`), covering the pure-Rust
mapping logic (always compiled) and the feature-gated import/export paths (compiled only
with `--features reqif`).

## Test Cases

### TC-001 — `can_handle` accepts ReqIF extensions
Verify that `ReqIfAdapter::can_handle` returns `true` for `.reqif` and `.reqifz` paths
and `false` for any other extension.

### TC-002 — `map_type` maps known spec-type strings
Verify that `"HLR"` → `RequirementType::Hlr`, `"LLR"` → `RequirementType::Llr`,
`"TST"` / `"TEST"` → `RequirementType::Tst`, and unknown strings default to `Llr`.

### TC-003 — `map_status` maps known status strings
Verify that `"approved"`, `"draft"`, `"deprecated"`, `"rejected"` round-trip correctly
through the default mapping. Unknown strings shall default to `Draft`.

### TC-004 — `ReqIfMapping::default()` contains expected attribute names
Verify that the default mapping uses the standard ReqIF attribute names
(`ReqIF.ForeignID`, `ReqIF.Content`, `ReqIF.Name`, `ReqIF.Status`).

### TC-005 — `read()` without feature returns config error
When compiled without `--features reqif`, calling `read()` on any path shall return
`Error::Config` with a message indicating the feature must be enabled.

### TC-006 — `read()` with feature rejects non-ReqIF extension  (feature-gated)
When compiled with `--features reqif`, calling `read()` on a file with an extension
other than `.reqif` / `.reqifz` shall return `Error::Parse`.

### TC-007 — `write()` without feature returns config error
When compiled without `--features reqif`, calling `write()` shall return `Error::Config`.

## Acceptance Criteria

- [ ] TC-001 through TC-005 pass in default build (`cargo test`)
- [ ] TC-006 passes when built with `--features reqif`
- [ ] TC-007 passes in default build
