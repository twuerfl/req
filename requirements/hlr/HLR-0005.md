## Description

The tool shall efficiently analyze large repositories.

## Rationale

Large codebases with millions of lines require efficient scanning. Methods like Hash-based caching or incremental scanning enables updates without full rescans.

## Acceptance Criteria

- [ ] Only changed files are re-parsed (Incremental)
- [ ] Hash-based caching is used
- [ ] Cache can be fully rebuilt from repository data
