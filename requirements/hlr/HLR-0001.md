## Description

The tool shall use the Git repository as the single source of truth for all requirements, traceability data, and configuration.

## Rationale

Reproducibility and auditability require that the complete system state can be reconstructed from version-controlled data. External databases introduce hidden state and break determinism, making audits and AI processing unreliable.

## Acceptance Criteria

- [ ] All requirements and metadata are stored in files within the repository
- [ ] No mandatory external database is required
- [ ] Repository state fully defines tool output
