## Description

The tool shall recognize code with inline requirement tags in source code and associate it with LLRs.

## Rationale

Bidirectional traceability is required for safety standards. Inline tagging keeps traceability close to implementation and reduces maintenance overhead.

## Acceptance Criteria

- [ ] Configurable regex-based tag detection
- [ ] File, symbol, and line references are stored
- [ ] Orphan code is detected
- [ ] Bidirectional traceability is derivable
